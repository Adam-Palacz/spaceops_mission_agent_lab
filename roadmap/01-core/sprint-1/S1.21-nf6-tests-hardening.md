# S1.21 — NF6 tests hardening (LLM/MCP mocking, no OPENAI dependency)

| Field | Value |
|-------|--------|

| **Status** | Done |

---

## Description

NF6 (limits/timeouts) behaviour is currently exercised in `tests/test_limits_timeouts.py`, but several tests depend on a real OpenAI Chat Completions call. In environments without `OPENAI_API_KEY` (including the default CI `test` job), those tests are skipped, so NF6 is only fully gated by the evals job. This task makes the NF6 tests self-contained by mocking LLM and MCP calls, so they always run (and fail on regressions) even when no external services or keys are available.

---

## Requirements

- [ ] **Remove hard dependency on `OPENAI_API_KEY` in NF6 tests:** Refactor `tests/test_limits_timeouts.py` so that tests for timeout/token/rate limits no longer skip when the key is missing.
- [ ] **Mock LLM calls:** In NF6 tests, monkeypatch `_chat_completion` (or the HTTP layer) to return deterministic content and token usage, without issuing real network calls.
- [ ] **Optional MCP mocking:** Where tests would otherwise depend on MCP servers, mock `call_telemetry` / `call_search_runbooks` / `call_search_postmortems` (or their call sites) to keep tests fast and local.
- [ ] **Keep evals as higher-level gate:** Evals job remains as an end-to-end integration gate, but NF6 invariants must already be enforced via unit tests in the standard `test` job.

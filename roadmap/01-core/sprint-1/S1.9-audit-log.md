---

## Description

Implement append-only audit log with schema from goals.md §4.6: timestamp, trace_id, incident_id, actor (agent | human), tool, args_hash, decision, policy_result, outcome. Storage: NDJSON file or DB table; process only appends, no in-place edits.

---

## Requirements

- [ ] Every agent action (tool call) produces one audit log entry.
- [ ] Schema: `timestamp`, `trace_id`, `incident_id`, `actor`, `tool`, `args_hash`, `decision`, `policy_result`, `outcome`.
- [ ] Append-only: no update/delete of existing entries (NDJSON append or INSERT-only table).
- [ ] Documented in goals.md §4.6; evals/ops can assume this format.

---

## Checklist

- [ ] Choose storage: single NDJSON file (e.g. `data/audit.ndjson`) or DB table with no UPDATE/DELETE.
- [ ] Implement logger/writer that appends one JSON object per event with all schema fields.
- [ ] Integrate with agent: after each MCP call (or node), write audit entry (actor=agent, tool=name, etc.).
- [ ] Trace_id and incident_id populated from run context.

---

## Test requirements

- [ ] After one agent run, audit log contains entries for each tool call; schema fields present.
- [ ] No code path that updates or deletes existing audit rows/lines.
- [ ] args_hash is consistent for same args (e.g. hash of canonical JSON).

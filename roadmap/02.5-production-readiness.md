# SpaceOps Mission Agent Lab - Phase 02.5 / Production Readiness

**Purpose:** turn the Production Scale reference system into an operationally credible production
pilot baseline before deep L3/L4 autonomy work. This phase is intentionally stronger than a minimum
bridge: it defines the reliability, observability, security, release, and operating gates that a
real production system would need before autonomous complexity increases.

Execution is tracked in [02.5-production-readiness/](02.5-production-readiness/).

---

## Position in the roadmap

| Phase | Role |
|-------|------|
| [02-production-scale/](02-production-scale/) | Production-like reference, cloud-stage proof, safety/eval foundation. |
| [02.5-production-readiness/](02.5-production-readiness/) | Production operations hardening and go/no-go gates. |
| [03-next-gen-autonomy/](03-next-gen-autonomy/) | L3/L4 autonomy features on top of a hardened operating base. |

NG1-NG2 may run in parallel when kept local or stage-lab scoped. NG3+ should not be promoted beyond
local proof until this phase's hard gates are complete.

---

## Themes

1. **Observability and SLOs:** in-cluster metrics, dashboards, alerts, trace/log retention, and
   health probes that operators can rely on.
2. **Reliability and data durability:** managed Postgres posture, backups, restore drills, failure
   tests, and long-lived stage policy.
3. **Security and compliance:** secret rotation, MCP/OPA/approval boundary review, audit integrity,
   redaction/privacy readiness.
4. **Release and operations:** smoke/e2e gates, rollback criteria, incident drills, on-call runbooks,
   and a production pilot go/no-go review.

---

## Hard exit criteria

- Stage is either long-lived or has an explicit ephemeral policy with recreate time, secret
  bootstrap, and demo readiness documented.
- Prometheus/Grafana or managed monitoring is deployed through Helm or GitOps for K8s stage.
- SLO dashboards and alert rules exist for API, agent worker, queue/DLQ, Postgres, LLM budget, and
  safety/eval regressions.
- Postgres backup and restore drill is executed and documented with recovery time and data-loss
  notes.
- Secrets are managed through the stage/prod path and rotated at least once.
- Release smoke/e2e gates run before stage/prod promotion.
- Security review covers OPA fail-closed, approvals auth, MCP boundary, audit append-only behavior,
  LLM gateway budget controls, and platform ops read-only/apply gating.
- A production readiness review records go/no-go status, residual risks, and owner decisions.

---

## Non-goals

- Connecting to real spacecraft or real ground segment systems.
- Claiming unattended 24/7 mission operations without an approved production pilot.
- Making live multi-cloud burst a prerequisite; it remains a later resilience option.
- Expanding autonomy features unless their production gates are explicit.

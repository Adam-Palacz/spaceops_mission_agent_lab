## Description

Create or clone an `ops-config` repo (or subtree in this repo) with sample config (e.g. alert thresholds YAML). This is the target for GitOps PRs from the agent (S2.2, S2.6).

---

## Requirements

- [ ] ops-config exists and is reachable (separate repo or subtree under repo root).
- [ ] Contains at least one sample config file (e.g. `alerts/thresholds.yaml` or similar).
- [ ] GitOps MCP can open PRs against it (branch protection optional for MVP).

---

## Checklist

- [ ] Create new repo `ops-config` or folder `ops-config/` in this repo.
- [ ] Add sample YAML (e.g. alert thresholds, channel list); commit.
- [ ] Document location and how agent/PR will target it (branch name, path).
- [ ] If separate repo: document clone URL and auth (e.g. token for GitHub API).

---

## Test requirements

- Repo/folder is in place; GitOps MCP (S2.2) can read and create branches/PRs against it.
- Sample config is valid and human-readable.

## Implementation notes

- **Metrics (`apps/api/main.py`):**
  - `/metrics` endpoint exposes Prometheus text format using `prometheus_client`.
  - Counters/histograms:
    - `agent_runs_total{status="success|error"}` — incremented for every `/runs` call.
    - `agent_run_duration_seconds` — observes wall-clock duration of `run_pipeline`.
    - `agent_errors_total{type="ExceptionClass"}` — incremented on exceptions in `/runs`.
    - `agent_tool_calls_per_run` — observes `llm_calls_used` from agent result per run (MoP2 proxy).
- **Infra (`infra/docker-compose.yml`):**
  - `prometheus` service (image `prom/prometheus:v3.10.0`) mounting `infra/prometheus.yml`, with `extra_hosts: host.docker.internal` so it can scrape the host API on `host.docker.internal:8000/metrics`.
  - `grafana` service (image `grafana/grafana:12.4.0`) on `http://localhost:3000`, provisioning directory `infra/grafana/provisioning` (datasource and dashboard).
- **Config (to be added when wiring dashboards):**
  - `infra/prometheus.yml` — scrape config for `spaceops-api` (`host.docker.internal:8000`).
  - `infra/grafana/provisioning/datasources/prometheus.yml` — Prometheus datasource (`http://prometheus:9090`).
  - Optional JSON dashboard under `infra/grafana/provisioning/dashboards/` with panels for runs, latency, errors, and tool-call histogram.

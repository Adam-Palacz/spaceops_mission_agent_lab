# S2.8 — Injection suite

| Field | Value |
|-------|--------|
| **Task ID** | S2.8 |
| **Status** | Todo |

---

## Description

Create `evals/injection_suite/`: 5–10 documents (e.g. fake runbooks) that attempt to trigger unsafe or off-policy actions (e.g. "ignore OPA", "run shell command"). Evals must fail if the agent performs any unsafe action (unsafe-action rate = 0). Goals: NF4, MoE3 (goals.md §4.4).

---

## Requirements

- [ ] At least 5–10 documents in `evals/injection_suite/` that try to manipulate the agent (prompt injection, fake instructions).
- [ ] Documents are indexed in KB or used as context so the agent could see them during evals.
- [ ] Eval cases that use these docs: expected outcome = agent must NOT perform unsafe action (e.g. no shell, no bypass OPA, no forbidden tool).
- [ ] CI evals: if agent performs unsafe action in any injection case → fail (MoE3 = 0).
- [ ] Scoring or assertion: unsafe_action_performed = false for all injection cases.

---

## Checklist

- [ ] Write 5–10 injection docs (markdown or text); place in `evals/injection_suite/`.
- [ ] Add eval cases that load these docs (e.g. into KB or as context) and run agent; expected = no unsafe action.
- [ ] Implement check in scoring: detect unsafe action (e.g. tool allowlist violation, shell call, OPA bypass); fail if any.
- [ ] Add CI step or extend existing evals job to run injection suite and enforce 0 unsafe actions.
- [ ] Document what "unsafe" means (reference goals NF1, NF3, P1–P3).

---

## Test requirements

- Running injection evals: all cases pass when agent does not perform unsafe action.
- If agent is modified to perform unsafe action in one case, injection eval fails (CI red).
- No false positives: normal runbook content does not trigger unsafe flag.

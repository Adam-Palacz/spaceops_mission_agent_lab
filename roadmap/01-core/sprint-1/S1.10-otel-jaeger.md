---

## Description

Instrument agent and FastAPI with OpenTelemetry (traces, logs). Export to OTel Collector; traces visible in Jaeger. Structured logging (JSON or key-value). Production-ready: structured logging (goals.md §4.5); NF2 (all agent actions in traces).

---

## Requirements

- [ ] Agent: create spans for each node (Triage, Investigate, Decide, Report) and for MCP tool calls.
- [ ] FastAPI: spans for requests (e.g. /ingest, /health, run trigger).
- [ ] Export to OTel Collector (configured in docker-compose S1.2); Collector forwards to Jaeger.
- [ ] Logs: structured (JSON or key-value), not free-form only.
- [ ] Report includes link to trace (Jaeger trace URL) — coordinate with S1.7.

---

## Checklist

- [ ] Add OTel SDK to agent and API; create TracerProvider, export to OTLP (Collector).
- [ ] In agent: start span per node and per tool call; set attributes (incident_id, tool name, etc.).
- [ ] In API: middleware or decorator to create span per request.
- [ ] Configure logging to output JSON or key-value; attach trace_id to log lines if possible.
- [ ] Build Jaeger trace URL from trace_id and put in Report (S1.7).

---

## Test requirements

- [ ] Full agent run produces one trace in Jaeger with spans for Triage, Investigate, Decide, Report and tool calls.
- [ ] Report contains a clickable or copy-paste Jaeger URL that opens the correct trace.
- [ ] Logs contain trace_id for correlation.

## Description

Extend Decide node: output steps tagged **safe** vs **restricted**. Act node: execute safe steps (call Ticketing or GitOps MCP); for restricted steps, call OPA (S2.4) and if allow create approval request (S2.5). Goals: F4, F5, F6.

---

## Requirements

- [ ] Decide output: list of steps; each step has type (safe | restricted) and action (e.g. create_ticket, create_pr, change_config).
- [ ] Act node: for each safe step, call corresponding MCP (Ticketing or GitOps); for restricted, do not execute directly — call OPA then create approval request if allowed.
- [ ] Safe steps execute in same run; restricted steps only after human approval (S2.6).
- [ ] Citation grounding (NF5a) still applies to plan steps.

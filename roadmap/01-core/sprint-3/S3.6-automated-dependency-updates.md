# S3.6 — Automated dependency updates (Dependabot/Renovate)

| Field | Value |
|-------|--------|
| **Task ID** | S3.6 |
| **Status** | Done |

---

## Description

Set up **automated dependency update tooling** (e.g. Dependabot or Renovate) so that Python
and other dependencies are regularly proposed for update, and ensure that updates are gated
by tests and evals. Category: **Infra/Sec tech debt** (dependency drift).

---

## Requirements

- [x] Choose a tool (Dependabot, Renovate, or equivalent) suitable for the repository host
      (e.g. GitHub).
- [x] Configure it to:
  - [x] watch Python dependencies (e.g. `requirements.txt`, `infra` images if desired),
  - [x] propose updates at a reasonable cadence (e.g. weekly).
- [x] Ensure CI pipelines (lint, tests, evals) run automatically on dependency update PRs.
- [x] Document how to review and merge such PRs (incl. what to check in evals).

---

## Checklist

- [x] Add configuration file for the chosen tool (e.g. `.github/dependabot.yml` or `renovate.json`).
- [x] Verify that at least one dependency update PR is opened in a test run or via dry-run.
- [x] Confirm CI runs all relevant checks (ruff, mypy, pytest, evals) on dependency PRs.
- [x] Document the policy: e.g. “do not merge dependency PRs with failing evals, even if
      unit tests pass”.

---

## Test requirements

- [x] A sample dependency update PR goes through CI and only merges when all checks pass.
- [x] If evals detect a behavioural regression after a dependency bump, the PR remains blocked
      until the issue is fixed or the update is adjusted.

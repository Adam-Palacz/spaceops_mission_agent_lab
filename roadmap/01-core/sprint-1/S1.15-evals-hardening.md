## Description

Evals currently can pass CI while not enforcing MoE2 (citation presence) or correct top-k semantics: `expected_subsystem_top_k` is present in cases but unused in scoring; when `require_citations: true`, an escalated run (e.g. no MCP/KB) is allowed to pass, so "citation-present" can pass without real citations. This task fixes scoring so evals actually gate quality.

---

## Requirements

- [x] **Use `expected_subsystem_top_k` in scoring:** Allow only the first k subsystems from `expected_subsystem` (ordered). E.g. top_k=1 ⇒ actual must be in `expected_subsystem[:1]`; top_k=2 ⇒ actual in `expected_subsystem[:2]`. Ensure `cases.yaml` order is [top1, top2, ...] where needed.
- [x] **Citation vs escalation:** If a case has `require_citations: true` and **not** `must_escalate: true`, then escalation must **fail** the case (e.g. "expected citations but got escalation"). Do not allow escalation to "rescue" a citation-required case.
- [x] Evals CI remains deterministic; no new flakiness.

## Checklist

- [x] In `evals/scoring.py`: when `require_citations` is true and `must_escalate` is false, if `escalated` is true then append failure (e.g. "require_citations: expected citations but run escalated").
- [x] Run full evals locally and in CI; confirm citation-present fails when MCP/KB unavailable and run escalates, and triage cases respect top_k.

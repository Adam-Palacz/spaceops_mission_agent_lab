# S2.4 — OPA: allowlist + argument validation, fail-closed

| Field | Value |
|-------|--------|

| **Status** | Todo |

---

## Description

Add OPA (Docker or sidecar): policy bundle with tool allowlist and argument validation (e.g. forbid "restart all", limit time range). Expose `POST /v1/data/agent/allow` (input: action; output: allow/deny). **Fail-closed:** on OPA error, timeout, or unavailable → treat as deny and produce escalation packet (NF8). Goals: NF3, NF8, P2, P3, P6.

---

## Requirements

- [ ] OPA service runnable (docker-compose or sidecar); policy in Rego.
- [ ] Endpoint or API: input = action (tool name + args); output = allow or deny.
- [ ] Policy: allowlist of tool names; reject forbidden args (e.g. "restart all", overly broad time range).
- [ ] Fail-closed: if OPA unreachable, timeout, or returns error → application treats as **deny** and triggers escalation packet; no execution.
- [ ] Agent (Act node) calls OPA before executing restricted action; on deny or error → no approval request, escalation.

---

## Checklist

- [ ] Add OPA to infra (docker-compose or separate process); load policy bundle.
- [ ] Write Rego: allow list of tools; rule to deny "restart all" and similar; optional time-range check.
- [ ] In Act node: before restricted action, call OPA; on allow → create approval request; on deny/error/timeout → deny + escalation, do not execute.
- [ ] Document policy location and how to add rules (for S2.10 tests).

---

## Test requirements

- [ ] OPA allow for allowed tool + valid args; OPA deny for forbidden args.
- [ ] When OPA is down or times out: agent does not execute restricted action; escalation packet produced (or logged deny).
- [ ] S2.10: unit tests for OPA policies (allowlist, forbidden args, fail-closed behaviour).

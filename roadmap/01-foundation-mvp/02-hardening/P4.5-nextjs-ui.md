## Description

Optional Next.js app: incident list, approval request list, Approve/Reject buttons with auth (same as API key/token as backend). Improves operator experience; not required for MVP but listed in roadmap Phase 4.

---

## Requirements

- [ ] Next.js app (e.g. under `apps/ui/`) or separate repo; can run locally and call FastAPI.
- [ ] Pages: list incidents (from API), list approval requests (GET /approvals), Approve/Reject buttons that call POST /approvals/:id/approve and reject with auth.
- [ ] Auth: pass API key or token in requests to FastAPI (same as NF9); no separate user DB required for MVP.
- [ ] Read-only views OK for incidents; approval flow must be functional.
- [ ] Document how to run UI and set API base URL and auth.

---

## Checklist

- [ ] Scaffold Next.js app; add config for API base URL and auth token.
- [ ] Page: fetch and display incidents (if API exposes list).
- [ ] Page: fetch GET /approvals; display pending; buttons Approve/Reject call API with auth.
- [ ] Handle loading and error states; optional: success toast after approve/reject.
- [ ] README or docs: how to run UI, env vars (API_URL, API_KEY or AUTH_TOKEN).
- [ ] If skipped: mark task Blocked or Done with note "Deferred"; remove from active BOARD or leave as Todo.

---

## Test requirements

- UI loads; incidents and approvals load from API when configured.
- Clicking Approve on a pending request calls API and updates list (and request is executed per S2.6).
- Unauthenticated or wrong token: API returns 401; UI shows error or does not expose actions.

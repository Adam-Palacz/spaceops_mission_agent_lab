## Description

Add FastAPI app in `apps/api`: `GET /health`, `POST /ingest` (accept NDJSON, validate, persist to data/ or DuckDB), and an endpoint or CLI to trigger an agent run (e.g. POST with incident payload or `incident_<id>.json`). Align with goals F1 (Ingest) and roadmap S1 outcomes.

---

## Requirements

- [ ] `GET /health` returns 2xx and indicates service is up.
- [ ] `POST /ingest` accepts NDJSON body; validates schema; persists to `data/` or DuckDB.
- [ ] Trigger run: e.g. `POST /runs` with incident payload or similar; triggers agent run (agent implementation in S1.7).
- [ ] App runnable via uvicorn (or similar); documented in README (S1.13).

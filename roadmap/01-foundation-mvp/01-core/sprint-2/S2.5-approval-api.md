# S2.5 — Approval API (idempotent, auth)

| Field | Value |
|-------|--------|

---

## Description

Implement approval API: `GET /approvals`, `POST /approvals/:id/approve`, `POST /approvals/:id/reject`. Idempotent (repeated approve same id = same result); require authentication (API key or token in MVP); record who approved/rejected and when in audit (NF9). Goals: F8, NF9.

---

## Requirements

- [x] `GET /approvals` returns list of pending (and optionally all) approval requests.
- [x] `POST /approvals/:id/approve` and `POST /approvals/:id/reject` update request status; require auth (e.g. API key in header or Bearer token).
- [x] Idempotent: calling approve again for already-approved id returns success and does not re-execute (S2.6).
- [x] Audit: each approve/reject writes to audit log with actor=human, who (identity from auth), when, decision, outcome.
- [x] Storage: approval_requests table or file; status = pending | approved | rejected.

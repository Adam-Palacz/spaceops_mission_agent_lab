## Description

Add 5–10 eval cases: triage top-1/top-2 correct, citation present, and at least one “must escalate” case. Implement `evals/scoring.py`; add CI job to run evals. Deterministic (pinned model or seed). Goals: NF4, MoE1–MoE4 (goals.md §4.4).

---

## Requirements

- [ ] Cases in `evals/cases.yaml` (or similar): inputs (e.g. incident payload), expected outputs (subsystem, citations, or “must escalate”).
- [ ] Scoring: triage correctness (top-1 and/or top-2), citation presence, escalation when required.
- [ ] Deterministic: same case yields same result (pin model or seed).
- [ ] CI job (e.g. GitHub Actions) runs evals on push/PR; fail if score below threshold or “must escalate” not satisfied.
- [ ] At least one case that must trigger escalation (low confidence or no evidence).

---

## Checklist

- [ ] Define case format (YAML: input, expected_subsystem, expected_citations, or must_escalate).
- [ ] Add 5–10 cases covering triage, citation, and one must-escalate.
- [ ] Implement `evals/scoring.py`: run agent per case, compare to expected, compute score.
- [ ] Add CI workflow step: install deps, run evals, enforce pass threshold.
- [ ] Document how to add a case (can be in Phase 4 runbook).

---

## Test requirements

- [ ] `python evals/scoring.py` (or equivalent) runs and outputs score/pass per case.
- [ ] CI runs evals and fails when a case fails or threshold not met.
- [ ] Must-escalate case: when input has low confidence/no evidence, agent produces escalation packet and eval passes.

# S2.7 — Audit log: approval events

| Field | Value |
|-------|--------|
| **Task ID** | S2.7 |
| **Status** | Todo |

---

## Description

Ensure audit log includes approval events: when a human approves or rejects, write entry with actor=human, tool (or action type), decision, outcome. Same schema as S1.9 (goals.md §4.6); append-only.

---

## Requirements

- [ ] Every approve/reject (S2.5) produces one audit log entry.
- [ ] Fields: timestamp, trace_id, incident_id, actor=human, tool (or approval_request_id), args_hash (optional), decision (approve/reject), policy_result (N/A or allow), outcome (success/skipped).
- [ ] Who approved/rejected: store in audit or in approval_requests; link to audit entry (e.g. decided_by user id or token id).
- [ ] Append-only; no deletion or update of audit rows.

---

## Checklist

- [ ] In S2.5 approve/reject handler: after updating approval_requests, call audit logger with actor=human, decision, outcome, identity.
- [ ] Ensure decided_at and decided_by (or equivalent) are stored and visible for compliance.
- [ ] Optional: add approval_request_id to audit entry for correlation.

---

## Test requirements

- After approve and reject, audit log contains two entries (one per action) with actor=human and correct decision/outcome.
- Schema matches goals.md §4.6; no new required fields missing.

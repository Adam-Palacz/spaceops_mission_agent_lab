## Description

Add OPA (Docker or sidecar): policy bundle with tool allowlist and argument validation (e.g. forbid "restart all", limit time range). Expose `POST /v1/data/agent/allow` (input: action; output: allow/deny). **Fail-closed:** on OPA error, timeout, or unavailable → treat as deny and produce escalation packet (NF8). Goals: NF3, NF8, P2, P3, P6.

---

## Requirements

- [x] OPA service runnable (docker-compose or sidecar); policy in Rego.
- [x] Endpoint or API: input = action (tool name + args); output = allow or deny.
- [x] Policy: allowlist of tool names; reject forbidden args (e.g. "restart all", overly broad time range).
- [x] Fail-closed: if OPA unreachable, timeout, or returns error → application treats as **deny** and triggers escalation packet; no execution.
- [x] Agent (Act node) calls OPA before executing restricted action; on deny or error → no approval request, escalation.

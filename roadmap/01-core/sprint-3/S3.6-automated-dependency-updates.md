# S3.6 — Automated dependency updates (Dependabot/Renovate)

| Field | Value |
|-------|--------|
| **Task ID** | S3.6 |
| **Status** | Todo |

---

## Description

Set up **automated dependency update tooling** (e.g. Dependabot or Renovate) so that Python
and other dependencies are regularly proposed for update, and ensure that updates are gated
by tests and evals. Category: **Infra/Sec tech debt** (dependency drift).

---

## Requirements

- [ ] Choose a tool (Dependabot, Renovate, or equivalent) suitable for the repository host
      (e.g. GitHub).
- [ ] Configure it to:
  - [ ] watch Python dependencies (e.g. `requirements.txt`, `infra` images if desired),
  - [ ] propose updates at a reasonable cadence (e.g. weekly).
- [ ] Ensure CI pipelines (lint, tests, evals) run automatically on dependency update PRs.
- [ ] Document how to review and merge such PRs (incl. what to check in evals).

---

## Checklist

- [ ] Add configuration file for the chosen tool (e.g. `.github/dependabot.yml` or `renovate.json`).
- [ ] Verify that at least one dependency update PR is opened in a test run or via dry-run.
- [ ] Confirm CI runs all relevant checks (ruff, mypy, pytest, evals) on dependency PRs.
- [ ] Document the policy: e.g. “do not merge dependency PRs with failing evals, even if
      unit tests pass”.

---

## Test requirements

- [ ] A sample dependency update PR goes through CI and only merges when all checks pass.
- [ ] If evals detect a behavioural regression after a dependency bump, the PR remains blocked
      until the issue is fixed or the update is adjusted.

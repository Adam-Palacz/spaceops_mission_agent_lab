# S2.10 — OPA unit tests

| Field | Value |
|-------|--------|
| **Task ID** | S2.10 |
| **Status** | Todo |

---

## Description

Add unit tests for OPA policies: allowlist (allowed tools pass, disallowed fail), argument validation (forbidden args denied), and fail-closed behaviour (when OPA returns error or timeout, application denies). Goals: NF3, P2, P3, P6; regression safety.

---

## Requirements

- [ ] Tests for policy allowlist: e.g. allowed tool + valid args → allow; disallowed tool → deny.
- [ ] Tests for argument validation: e.g. "restart all" → deny; valid narrow args → allow (if in allowlist).
- [ ] Tests for fail-closed: mock OPA timeout/error; application code must treat as deny (and optionally escalation); no execution.
- [ ] Tests run in CI (pytest or equivalent); no flake.
- [ ] OPA policy files are versioned; tests load same policy as runtime.

---

## Checklist

- [ ] Create test file(s) for OPA (e.g. `tests/policy/test_opa_allowlist.py` or Rego tests).
- [ ] Test cases: allow listed tool; deny unlisted tool; deny "restart all"; deny oversized time range (if in policy).
- [ ] Test application layer: when OPA client returns error or timeout, Act node (or policy client) returns deny and does not execute.
- [ ] Add test run to CI (pytest); document how to run locally.

---

## Test requirements

- All OPA policy tests pass; CI runs them.
- Changing policy to allow "restart all" causes a test to fail (regression).
- Fail-closed test: simulated OPA failure results in deny path in application.

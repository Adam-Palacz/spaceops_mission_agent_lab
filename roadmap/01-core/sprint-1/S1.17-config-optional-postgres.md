# S1.17 — Config: optional Postgres for no-DB modes

| Field | Value |
|-------|--------|
| **Task ID** | S1.17 |
| **Status** | Todo |

---

## Description

`postgres_password` in `config.Settings` is required (no default), which forces a value even when running "API only" or "Telemetry MCP only" without the KB server or indexer. This task relaxes config so DB-related settings are only required when the app actually uses Postgres (e.g. KB server, index_kb).

---

## Requirements

- [ ] **Make `postgres_password` optional** for the core app: e.g. default `""` (or a sentinel). When building `postgres_dsn` / using DB, either require the password (e.g. raise or skip) or document that KB/indexer need `POSTGRES_PASSWORD` set.
- [ ] **No breaking change for current usage:** If `POSTGRES_PASSWORD` is set (or `DATABASE_URL` is set), behaviour unchanged. Only "no DB" runs (API + agent without KB, or Telemetry MCP only) work without setting a dummy password.
- [ ] KB server and `index_kb` should fail fast with a clear error if Postgres is not configured (e.g. missing password or URL), rather than failing with a low-level connection error later.

---

## Checklist

- [ ] In `config.py`: give `postgres_password` a default (e.g. `Field(default="")`); ensure `postgres_dsn` is only used when DB is actually needed (KB server, indexer) or document that `postgres_dsn` may be invalid when password is empty.
- [ ] In apps that use Postgres (e.g. `apps/mcp/kb_server/`, `index_kb`): on startup or first use, check that DB config is present (e.g. non-empty password or DATABASE_URL) and raise a clear error if not.
- [ ] Update `.env.example` to note that Postgres vars are optional when not using KB/RAG.

---

## Test requirements

- [ ] Test: API + agent (without KB) start and run one pipeline without `POSTGRES_PASSWORD` set (or with empty); no connection attempt to Postgres from core app.
- [ ] Test: KB server or `index_kb` started without valid Postgres config (e.g. empty password, no DATABASE_URL) raise a clear error at startup or first use, not a generic connection error.
- [ ] Test: with `POSTGRES_PASSWORD` (or DATABASE_URL) set, existing behaviour unchanged; pytest and evals pass.

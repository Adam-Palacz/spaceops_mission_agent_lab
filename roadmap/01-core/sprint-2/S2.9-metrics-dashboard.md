## Description

Add Prometheus metrics: `agent_runs_total`, `agent_run_duration_seconds`, `agent_errors_total`, tool-call count per run. Add Grafana to docker-compose; create dashboard with run count, latency, error rate, tool-call count. Goals: MoP1, MoP2 (goals.md §4.4); observability.

---

## Requirements

- [ ] Agent (or API) exposes Prometheus metrics: runs total, run duration, errors total; optionally tool_calls_total or per-run histogram.
- [ ] Prometheus scrapes the agent/API; add Prometheus service to docker-compose if not present.
- [ ] Grafana in docker-compose; datasource = Prometheus.
- [ ] Dashboard: at least run count, mean/percentile latency, error rate, tool-call count (or max per run).
- [ ] Metrics available after a few runs; dashboard loads without error.

---

## Checklist

- [ ] Add Prometheus client to agent or API; increment counters and observe duration per run.
- [ ] Add tool-call counter (per run or global); expose in same endpoint.
- [ ] Add Prometheus to infra (docker-compose); scrape config for agent/API.
- [ ] Add Grafana service; provision Prometheus datasource; create dashboard panels.
- [ ] Document how to open Grafana (e.g. http://localhost:3000) and what each panel shows.

---

## Test requirements

- [ ] After N agent runs, Prometheus shows agent_runs_total >= N; duration and error metrics present.
- [ ] Grafana dashboard displays data; no scrape or query errors.
- [ ] Tool-call count visible (per run or total) for MoP2.

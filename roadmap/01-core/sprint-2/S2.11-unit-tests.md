## Description

Add or expand **unit tests in tests/** for Sprint 2 scope: approval API (GET /approvals, approve/reject, idempotency, auth), OPA client (allow/deny/error handling), Act execution path (safe vs restricted), and optionally MCP Ticketing/GitOps with mocks. S2.10 covers OPA policy (Rego) tests; this task covers application code in **tests/** run by pytest. Ensures regression safety and documents behaviour.

---

## Requirements

- [ ] **tests/** structure extended for S2 (e.g. `tests/api/` for approval endpoints, `tests/agent/` or `tests/unit/` for Act/OPA client).
- [ ] Unit tests for approval API: GET /approvals returns list; approve/reject require auth (401 without); idempotent (second approve same id does not re-execute); audit entry created with actor=human.
- [ ] Unit tests for OPA client/caller: when OPA returns allow → proceed; when deny/error/timeout → deny path and no execution (fail-closed).
- [ ] Optional: tests for Act node (safe step calls MCP; restricted step calls OPA then creates request or escalates); use mocks for MCP and OPA.
- [ ] All tests run with pytest; CI runs them; deterministic (mocked external services).
- [ ] No duplication of S2.10 (OPA policy Rego tests); S2.11 focuses on app logic that calls OPA and approval API.

# S1.22 — Code style enforcement (ruff format, config/infra readability)

| Field | Value |
|-------|--------|

| **Status** | Todo |

---

## Description

S1.19 introduced an initial readability pass over core agent/evals modules, but several important files (agent nodes, CI workflow, Docker Compose, and other infra/config) still use a compressed, one-liner-heavy style. This makes review and debugging harder as the system grows in S2 (Act, OPA, approvals). This task adds an explicit style/formatting baseline (ruff format + light manual fixes) so that Sprint 1 code matches a “professional” readability standard and future changes can rely on automated formatting.

---

## Requirements

- [ ] **Enable/standardise auto-formatting:** Adopt `ruff format` (or equivalent) as the default formatter for Python code in this repo; document how to run it locally and/or wire it into pre-commit.
- [ ] **Tidy core agent modules:** Ensure `apps/agent/nodes.py`, `apps/agent/graph.py`, `apps/agent/mcp_client.py`, and `evals/scoring.py` have consistent, non-compressed formatting (no dense multi-statement one-liners where clarity suffers).
- [ ] **Improve infra/config readability:** Do a light readability pass over `infra/docker-compose.yml`, `.github/workflows/ci.yml`, and key config/docs so that conditionals, env vars, and steps are easy to scan.
- [ ] **Keep changes behaviour-neutral:** No refactors or logic changes; only line breaks, naming where helpful, and formatting.

---

## Checklist

- [ ] Decide on the canonical formatter (ruff format) and:
  - [ ] Add a note to README/docs on running `ruff format .` locally.
  - [ ] Optionally add a `ruff format` hook to `.pre-commit-config.yaml`.
- [ ] Run formatting + small manual cleanups over:
  - [ ] `apps/agent/nodes.py`, `apps/agent/graph.py`, `apps/agent/mcp_client.py`.
  - [ ] `evals/scoring.py`, `config.py` (where helpful).
  - [ ] `infra/docker-compose.yml`, `.github/workflows/ci.yml` (split dense lines, align env vars/steps).
- [ ] Re-run `ruff check`, `mypy`, `pytest tests/`, and `python -m evals.scoring` to confirm no behavioural regressions.

---

## Test requirements

- [ ] CI remains green (lint, mypy, pytest, evals) after formatting changes.
- [ ] Spot-check review of key files confirms improved readability (fewer dense one-liners, clearer structure) without semantic changes.

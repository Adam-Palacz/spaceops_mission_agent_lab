| Field | Value |

---

## Description

Set up a **tests/** folder and add unit tests for Sprint 1 deliverables: API (health, ingest), audit log writer, fixture loading, and (where feasible) agent nodes or MCP clients. Tests run via pytest and are part of CI (S1.13). Keeps regression under control and documents expected behaviour.

---

## Requirements

- [ ] **tests/** exists at repo root (or under apps/ per app); structure is clear (e.g. `tests/api/`, `tests/agent/`, `tests/lib/` or `tests/unit/`).
- [ ] Unit tests for: `GET /health` and `POST /ingest` (validation, persistence); audit log append (schema, no update/delete); fixture loading or schema validation.
- [ ] Optional: tests for agent state schema, Triage/Investigate/Decide/Report with mocks (or integration-style with test fixtures).
- [ ] pytest runs all tests; CI (S1.13) runs `pytest tests/` (or equivalent); no flaky tests.
- [ ] Tests are deterministic (no reliance on live LLM or external services unless mocked).

---

## Checklist

- [ ] Create **tests/** and subdirs (e.g. `tests/unit/`, `tests/api/`, `tests/agent/`) as needed.
- [ ] Add `tests/conftest.py` or fixtures for shared test data (e.g. sample NDJSON, in-memory audit log).
- [ ] Add tests for API: health returns 200; ingest accepts valid NDJSON and rejects invalid; optional: trigger endpoint returns 202 or runs stub.
- [ ] Add tests for audit log: append adds one entry with required schema fields; no API to update/delete.
- [ ] Add tests for fixture schema or loader if used by ingest/agent.
- [ ] Ensure pytest discovers tests; document `pytest tests/` in README (S1.13). Add to CI job if not already covered by S1.13.

---

## Test requirements

- `pytest tests/` (or `pytest tests/unit/ tests/api/`) passes.
- New unit tests are run in CI; at least one test per critical path (health, ingest, audit log).
- Tests do not require Docker or live Postgres/LLM (use mocks or in-memory backends).

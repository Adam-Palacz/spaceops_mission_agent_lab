## Implementation notes

- **Client fail-closed tests** (`tests/test_opa_client.py`):
  - Simulate OPA responses: `result: true|false`, HTTP 5xx, timeout, connection error, malformed/empty JSON.
  - Assert `opa_allow()` returns `True` only when `result` is a boolean `True`; all error/edge cases → `False` (fail-closed).
- **Act node application tests** (`tests/test_act_opa_policy.py`):
  - When `opa_allow` returns `True` for a restricted step, `act()` creates an `approval_request` via `approval_store_create` and does not escalate.
  - When `opa_allow` returns `False` (policy deny or OPA failure), `act()` returns `escalated=True` with `reason="policy_deny"`, no approval_requests, and does not call `approval_store_create`.
- **Policy allowlist & \"restart all\" tests**:
  - Optional integration test in `tests/test_opa_client.py::test_opa_policy_allowlist_and_restart_all_deny_integration` uses the real Rego policy (`infra/opa/agent_policy.rego`) via the OPA HTTP API.
  - Test is enabled only when `OPA_POLICY_INTEGRATION=1` is set and OPA is running (e.g. `docker compose -f infra/docker-compose.yml up -d opa`), so CI can run it explicitly without flakiness.

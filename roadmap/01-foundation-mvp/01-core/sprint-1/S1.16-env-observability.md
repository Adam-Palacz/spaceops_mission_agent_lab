## Description

README describes OTel exporter and agent limits/timeouts, but `.env.example` does not list them. With an empty `otel_exporter_otlp_endpoint` by default, traces do not flow to Jaeger out-of-the-box even though Docker Compose exposes 4317/4318. This task makes "single-command run" actually show traces in Jaeger and documents all relevant env vars in one place.

---

## Requirements

- [x] **Extend `.env.example`** with:
  - `OTEL_EXPORTER_OTLP_ENDPOINT` (maps to `config.otel_exporter_otlp_endpoint`); suggest value for local compose (e.g. `http://localhost:4317`).
  - Agent limits/timeouts: `AGENT_RUN_TIMEOUT_SECONDS`, `AGENT_LLM_CALL_TIMEOUT_SECONDS`, `AGENT_TOKEN_BUDGET_PER_RUN`, `AGENT_MAX_LLM_CALLS_PER_RUN` (with short comments).

- [x] README (or .env.example comments) briefly state that setting OTLP endpoint enables trace export to the collector.

# S1.12 — Token/rate limits and timeouts (NF6)

| Field | Value |
|-------|--------|
| **Task ID** | S1.12 |
| **Status** | Done |

---

## Description

Add token/rate limits and timeouts for agent and LLM calls; when limits or timeout are hit, fallback to escalate-to-human (escalation packet). Prevents runaway loops and “frozen ops”. Goal NF6 (Must).

---

## Requirements

- [x] Token limit: cap total tokens per run or per node (configurable).
- [x] Rate limit: throttle LLM or MCP calls if needed (e.g. max N calls per minute).
- [x] Timeout: overall run timeout and/or per-node timeout; on timeout → escalation packet, no silent hang.
- [x] When any limit or timeout triggers: produce escalation packet (reuse S1.8), do not continue normal plan execution.
- [x] Configuration via env or config file (no hardcoding).

---

## Checklist

- [x] Add run-level and/or node-level timeout (e.g. 60s per run, 20s per LLM call).
- [x] Add token budget (e.g. max 50k tokens per run); track usage and stop + escalate when exceeded.
- [x] On timeout/limit: transition to escalation path (S1.8), set reason in escalation packet.
- [x] Document env vars or config keys for limits and timeouts.

---

## Test requirements

- [x] Run that exceeds timeout produces escalation packet and does not hang.
- [x] Run that exceeds token limit produces escalation packet.
- [x] Normal run under limits completes without false escalation.

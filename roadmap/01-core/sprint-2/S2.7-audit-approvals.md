# S2.7 — Audit log: approval events

| Field | Value |
|-------|--------|
| **Task ID** | S2.7 |
| **Status** | Done |

---

## Description

Ensure audit log includes approval events: when a human approves or rejects, write entry with actor=human, tool (or action type), decision, outcome. Same schema as S1.9 (goals.md §4.6); append-only.

---

## Requirements

- [x] Every approve/reject (S2.5) produces one audit log entry.
- [x] Fields: timestamp, trace_id, incident_id, actor=human, tool (or approval_request_id), args_hash (optional), decision (approve/reject), policy_result (N/A or allow), outcome (success/skipped).
- [x] Who approved/rejected: store in audit or in approval_requests; link to audit entry (e.g. decided_by user id or token id).
- [x] Append-only; no deletion or update of audit rows.

---

## Checklist

- [x] In S2.5 approve/reject handler: after updating approval_requests, call audit logger with actor=human, decision, outcome, identity.
- [x] Ensure decided_at and decided_by (or equivalent) are stored and visible for compliance.
- [x] Optional: add approval_request_id to audit entry for correlation.

---

## Test requirements

- [x] After approve and reject, audit log contains two entries (one per action) with actor=human and correct decision/outcome.
- [x] Schema matches goals.md §4.6; no new required fields missing.

---

## Implementation notes

- **API** (`apps/api/main.py`): Approve and reject handlers already called `audit_append` (S2.5). S2.7 adds `policy_result="n/a"` for human approval events and `approval_request_id` in args for correlation. `decided_by` is in args and stored in approval store (`decided_at`/`decided_by` in JSON).
- **Audit schema** (`apps/agent/audit_log.py`): Unchanged; entries include timestamp, trace_id, incident_id, actor, tool, args_hash, decision, policy_result, outcome.
- **Tests** (`tests/test_api.py`): `test_approvals_audit_entries_s27` — one approve + one reject on two requests; assert audit has ≥2 human entries with required schema keys and both decision approve and reject.
